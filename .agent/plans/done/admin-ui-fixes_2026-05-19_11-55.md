# Plano: Correção de UI do Painel Admin e Robustez do Banco

## Pedido do usuário
- Ao abrir a aba de configurações, estava ocorrendo "Erro ao carregar configuracoes".
- O botão "Salvar Legendas Globais" flutuante (fixed) fica sobreposto aos editores de texto, atrapalhando a usabilidade.

## Objetivo
1. Corrigir o layout do botão de salvar na `AdminConfigTab.tsx` para que ele flua naturalmente com a página, removendo sua propriedade `fixed`.
2. Assegurar que o `initServerConfig` no backend inicialize as novas propriedades como strings vazias em vez de permitir valores NULL na criação inicial do banco.

## Contexto atual
- O banco atualizou os esquemas, mas a linha já existente recebeu `NULL` em `global_default_caption` e `global_new_pack_caption`, causando possíveis desserializações nulas no frontend. (Manual fix was applied, but we need code robustness).
- A aba de admin usa uma div pai com o botão "fixed bottom-6", que sem o padding-bottom adequado, sobrepõe o conteúdo principal da aba.

## Arquivos analisados
- `internal/database/database.go`
- `dashboard/src/components/AdminConfigTab.tsx`

## Arquivos que poderão ser modificados
- `internal/database/database.go`
- `dashboard/src/components/AdminConfigTab.tsx`

## Estratégia de implementação
1. **Frontend:** Em `AdminConfigTab.tsx`, remover as classes `fixed`, `bottom-6`, `left-1/2`, `-translate-x-1/2`, e `z-50` do container do botão de salvar. Adicionar uma `div` com padding top (`pt-4`) para dar respiro.
2. **Backend:** Em `database.go`, na função `initServerConfig`, adicionar `GlobalDefaultCaption: ""` e `GlobalNewPackCaption: ""` no objeto `config` inicializado via `FirstOrCreate`.

## Passos detalhados
1. Atualizar o `AdminConfigTab.tsx` para posicionamento estático do botão.
2. Atualizar o `database.go` para inicialização completa do modelo.

## Riscos
- O botão pode ficar visualmente menos destacado se estiver no fim de uma rolagem muito longa, mas melhora 100% a acessibilidade e impede a sobreposição de conteúdo.

## Impactos esperados
- O administrador poderá rolar a tela, digitar o texto normalmente, e só então clicar no botão "Salvar" no final da página, sem nada bloqueando a visão.
- O banco será criado de forma segura em novas instalações, sem NULL values.

## Como testar
1. Abrir a Dashboard e ir na aba "Admin" -> "Configurações".
2. Rolar a página até o fim e observar o botão de salvar inserido harmoniosamente na página.
3. Testar a edição e confirmar se salva normalmente.
