# Plano: Painel de Telemetria (HUD) McLaren

## Pedido do usuário
O usuário concordou em substituir o antigo "Digital Access Card" (que tinha design de cartão de crédito e código de barras) por algo que combine mais com a temática McLaren de alta performance.

## Objetivo técnico
Criar um componente HUD (Heads-Up Display) de telemetria, substituindo o markup no `DashboardInicioTab.tsx` e atualizando o CSS correspondente no `index.css`.

## Estratégia de implementação

**1. Design Conceptual (Pit Wall Telemetry):**
- **Tema:** Painel técnico de engenheiro da McLaren.
- **Tipografia:** Uso de fontes monoespaçadas para rótulos e status.
- **Status Ativo:** Um indicador visual piscante (`pulse-dot`) simulando que o bot está "online" e transmitindo dados.
- **Identificação:** O usuário será exibido como `CHIEF ENGINEER`.
- **Acesso/Link:** O link do canal não será mais uma "raspadinha" (security scratch), mas um "SECURE UPLINK" integrado ao painel.
- **Visual:** O card terá o efeito Liquid-Glass e uma faixa lateral na cor Laranja Papaya (`--accent`) para identidade instantânea.

**2. Passos detalhados - Componente React (`DashboardInicioTab.tsx`):**
Substituir o bloco `<div className="access-card">...</div>` por `<div className="telemetry-hud">...</div>`.
A nova estrutura conterá:
- `.telemetry-header`: Contendo o `.pulse-dot` "SYSTEM ACTIVE" e o `ownerId`.
- `.telemetry-body`: Mostrando o nome do dono (`displayName`) com fonte de destaque.
- `.telemetry-footer`: Um painel interno menor para o "SECURE UPLINK" (invite link).

**3. Passos detalhados - Estilos CSS (`index.css`):**
- Remover todo o CSS legado do `.access-card`, `.card-chip`, `.card-barcode`, e `.security-scratch` (isso aliviará o arquivo CSS de animações complexas que não são mais usadas).
- Adicionar o bloco `/* ===== TELEMETRY HUD (McLAREN) ===== */`:
  - `.telemetry-hud`: Com backdrop blur (liquid glass), borda sutil, e borda lateral esquerda (border-left: 4px solid var(--accent)).
  - Animação `.telemetry-pulse` para o LED.
  - Estilização de terminais e fontes mono.

## Riscos
A substituição altera uma grande parte visual da tela inicial. O novo HUD precisa ser testado para garantir que links longos não quebrem o layout (usarei `truncate` e `overflow: hidden`). A limpeza do CSS precisa ser cuidadosa para não remover classes que outras partes do app possam estar utilizando acidentalmente (embora as classes mencionadas pareçam exclusivas deste componente).
