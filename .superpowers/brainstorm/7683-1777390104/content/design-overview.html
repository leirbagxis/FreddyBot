1. Core Data Flow

  • Authentication: Webapp sends initData to /api/auth. Server validates and sets a Secure JWT Cookie.
  • Channel Discovery: GET /api/me/channels populates the main dashboard.
  • Configuration: Users select a channel → fetch channel details → edit Captions/Buttons/Permissions → PUT updates.
  • Admin: GET /admin/api/users allows system-wide monitoring.

2. Proposed Component Map

  • ChannelList.tsx: Grid of channels owned by the user.
  • ChannelEditor.tsx: Tabbed interface for Captions, Buttons, and Permissions.
  • AdminPanel.tsx: Table of all users/channels for superusers.
  • api.ts: Cleaned up to match routes.go precisely.
